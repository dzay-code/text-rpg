# ⚔️ Text-Based QTE RPG Game

Консольная текстовая ролевая игра на языке Python, завязанная на механике **Quick Time Events (QTE)**. Игроку предстоит путешествовать по опасному лесу, сражаться с монстрами, проверять свою скорость реакции, собирать редкие артефакты и открывать достижения.

---

## 🚀 Основной функционал

* **🗺️ Поход (Основной режим):** Вылазки вглубь леса с возможностью выбора сложности и постепенного продвижения после победы над монстрами.
* **⚔️ Дуэль Вечности:** Бесконечный аркадный режим на выносливость, где цель — набрать как можно более длинную серию (стрик) успешных вводов подряд против выбранного типа монстра.
* **🎒 Система инвентаря и артефактов:** Уникальные трофеи, выпадающие с боссов (например, Сердце Лича, Вельзевул), которые можно активировать перед боем для получения преимуществ.
* **🥇 Достижения и Статистика:** Отслеживание убийств, смертей, соотношения K/D, а также гибкая система ачивок, сохраняемая в профиль игрока.
* **⚙️ Кастомизация испытаний:** Три режима ввода на выбор (**Буквы, Слова, Цифры**) и поддержка двух языков (**Русский / Английский**).
* **💾 Сохранение прогресса:** Автоматическая регистрация, авторизация и запись всех данных игрока в файлы формата JSON.

---

## 🎮 Игровой процесс и правила

1. **Суть боя:** На экране появляется случайная комбинация символов/слов. Игроку необходимо безошибочно ввести её и нажать `Enter` до того, как истечет выделенное на монстра время.
2. **Критический удар:** При каждом правильном вводе есть шанс нанести критический урон (снимает 2 единицы здоровья монстра вместо 1).
3. **Игровой лор и особенности существ:** Характеристики противников отражают их природу. Например, Берсерк обладает повышенным шансом впасть в ярость и урезать время на ввод, а Бессмертный Лич выделяется существенно большим запасом здоровья по сравнению с остальными монстрами.
4. **Побег и Засады:** В случае смертельного ранения у игрока есть небольшой шанс спастись бегством. Однако при попытке досрочно завершить поход или после побега есть риск попасть в засаду на обратном пути (при этом после побега шанс встретить нового монстра существенно выше).
5. **Динамические испытания:** Некоторые редкие противники (например, Апостол разработчика) способны прямо во время боя случайным образом изменять правила, на ходу переключая текущий режим ввода на любой другой из доступных в игре.

## 📂 Структура файлов проекта

* `rpg.py` — Главный исполняемый файл, содержащий логику меню, походов, дуэлей и боевых функций.
* `registration.py` — Модуль авторизации и создания учетных записей.
* `data.py` — Файл конфигурации базы данных монстров, предметов и словарей.
* `players.json` — Локальное хранилище данных всех зарегистрированных профилей.
* `last_user.json` — Файл сессии для автоматического входа последнего игрока.
